## 1. 基本ルール：毎回やること

### 作業開始時：最新の状態を取得

他の人が更新している可能性があるので、必ず最初に実行します。

```
git pull してください
```

### 作業終了時：変更を保存・共有

自分の変更を保存して、他の人も見れるようにします。

```
コミットしてプッシュしてください
```

または

```
変更を保存してプッシュしてください
```

> **ポイント**: 「コミット」「プッシュ」という言葉を使わなくても、「保存して共有して」でも伝わります。

## 2. テキストを変更したい

### キャッチコピー・タイトルを変更

```
ヒーローセクションのキャッチコピーを「あなたの未来を、ここから。」に変更してください
```

```
トップページのメインタイトルを「株式会社〇〇」に変えてください
```

### 説明文を変更

```
会社概要の説明文を以下に変更してください：
「私たちは創業50年の歴史を持つ〇〇です。」
```

### 複数箇所を一度に変更

```
以下のテキストを変更してください：
- ヒーローのタイトル → 「新しいタイトル」
- サブタイトル → 「新しいサブタイトル」
- ボタンのテキスト → 「お問い合わせ」
```

## 11. 確認・プレビュー


### 現在の状態を確認

```
今の変更内容を教えてください
```

```
git status を確認してください
```

### ビルドエラーの確認

```
ビルドしてエラーがないか確認してください
```

## 12. 困ったとき

### やり方がわからない

```
〇〇を変更したいのですが、どうすればいいですか？
```

```
このサイトで△△はどこにありますか？
```

### 元に戻したい

```
さっきの変更を元に戻してください
```

```
一つ前の状態に戻してください
```

### エラーが出た

```
エラーの内容を確認して、修正してください
```

```
ビルドエラーが出ているので直してください
```

### 何をすればいいかわからない

```
このプロジェクトの構造を教えてください
```

```
何から始めればいいですか？
```

## 指示のコツ

### 1. 具体的に伝える

**良い例**:
```
ヒーローセクションのタイトル「会社名」を「株式会社〇〇」に変更してください
```

**悪い例**:
```
タイトルを変えて
```

### 2. 場所を明確にする

**良い例**:
```
トップページのヒーローセクションの背景画像を変更してください
```

**悪い例**:
```
画像を変えて
```

### 3. 数値で指定できるものは数値で

**良い例**:
```
余白を40pxにしてください
```

または

```
余白を今の1.5倍にしてください
```

### 4. わからないことは聞く

```
〇〇を変更したいのですが、どのファイルを編集すればいいですか？
```

```
この部分の名前は何といいますか？
```

## よく使うフレーズ一覧

| やりたいこと | 指示例 |
|-------------|--------|
| 最新を取得 | `git pull してください` |
| 保存・共有 | `コミットしてプッシュしてください` |
| テキスト変更 | `〇〇を「△△」に変更してください` |
| 画像変更 | `〇〇の画像を △△.jpg に差し替えてください` |
| 大きくする | `〇〇を大きくしてください` |
| 小さくする | `〇〇を小さくしてください` |
| 余白を広げる | `〇〇の余白を広げてください` |
| 色を変える | `〇〇の色を #XXXXXX にしてください` |
| 追加する | `〇〇に△△を追加してください` |
| 削除する | `〇〇を削除してください` |
| 確認する | `〇〇を見せてください` |
| 元に戻す | `さっきの変更を元に戻してください` |
