## 目的

承認済みのHPを本番環境に公開し、先方に正式に納品する。運用方法についても説明する。

## フロー概要

```
【STEP 1】最終確認
    ↓ 納品物のチェック
問題なし
    ↓
【STEP 2】本番デプロイ
    ↓ サーバー移管 or DNS設定
公開URL確定
    ↓
【STEP 3】納品連絡
    ↓ 納品物一覧と運用説明
先方確認
    ↓
【STEP 4】運用説明
    ↓ 更新方法・問い合わせ先
運用開始
    ↓
ステータス更新
```

## 作業手順

### STEP 1: 最終確認

**チェックリスト:**
- [ ] 全ページが正しく表示される
- [ ] 画像がすべて表示される
- [ ] リンクが正しく動作する
- [ ] お問い合わせフォームが動作する
- [ ] メタタグ（title, description）が設定されている
- [ ] OGP画像が設定されている
- [ ] favicon.icoが設定されている
- [ ] ビルドエラーがない

**ビルド確認:**
```bash
npm run build
npm run start  # 本番モードで確認
```

---

### STEP 2: 本番デプロイ

#### パターンA: Vercelに本番デプロイ

1. **カスタムドメインを設定**
   - Vercelダッシュボード → Settings → Domains
   - ドメインを追加

2. **DNS設定を変更**
   - ドメイン管理画面（お名前.com等）を開く
   - AレコードまたはCNAMEをVercelに向ける

3. **SSL証明書を確認**
   - Vercelが自動でSSL証明書を発行
   - HTTPSでアクセスできることを確認

#### パターンB: CNAMEのみ変更

1. **現在のDNS設定を記録**
   - 変更前の設定をスクショ保存

2. **CNAMEを設定**
   - Vercelのデフォルトドメインに向ける
   - MXレコード（メール）は絶対に変更しない

3. **浸透を待つ**
   - 数時間〜最大48時間

#### パターンC: ビルド済みファイル納品

1. **静的ファイルを生成**
   ```bash
   npm run build
   npm run export  # 静的出力（必要に応じて）
   ```

2. **outフォルダを納品**
   - zipファイルにして共有

## トラブルシューティング

### DNS設定が反映されない場合
- TTL（キャッシュ時間）を確認
- 24〜48時間待つ
- DNSキャッシュをクリア

### SSLエラーが出る場合
- Vercelの証明書発行を待つ（数分〜数時間）
- ドメイン設定を再確認
