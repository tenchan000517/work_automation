## 1. 概要

### 1.1 コンセプト

中小企業向けの採用HP制作サービス。テンプレートベースで効率的に制作しながら、企業の特性に合わせたカスタマイズを行う。

**3つの柱:**
- **共感採用**: 社員のストーリーで「ここで働く自分」をイメージさせる
- **SEO/LLMO最適化**: 検索エンジン＋AI検索の両方に対応
- **モバイルファースト**: 応募者の70%以上がスマホからアクセス

### 1.2 業務数と担当者

| 項目 | 内容 |
|------|------|
| 総業務数 | 8業務 |
| 主担当 | 河合（構成設計・デザイン・実装・確認依頼・月次FB）、川崎（撮影） |
| CC | 青柳（全体管理・進捗確認） |
| 期間目安 | 受注〜公開まで約4週間 |

### 1.3 フェーズ構成

| Phase | 名称 | 業務 |
|-------|------|------|
| 0 | 受注・立ち上げ | 受注・初回ヒアリング |
| 1 | ヒアリング・企画 | 構成設計・テンプレート選定 |
| 2 | 素材準備 | 素材撮影 |
| 3 | デザイン | デザイン作成 |
| 4 | 実装 | コーディング |
| 5 | 確認・納品 | 確認依頼・キックオフMTG |
| 6 | 運用 | HP内容更新・月次FB |

## 2. 全体フロー図

```
【Phase 0: 受注・立ち上げ】
① 受注・ワークス立ち上げ（営業）
   ▼ [トリガー] ワークスで担当者メンション + 必要情報フォーマット送信
② 初回ヒアリング日程調整（営業）
   ▼ [トリガー] 日程確定報告（カレンダー更新）
③ 初回ヒアリング（営業, 河合 cc:青柳）
   ▼ [トリガー] 議事録のワークスグループ共有

【Phase 1: ヒアリング・企画】
④ 構成設計・テンプレート選定（河合 cc:青柳）
   ▼ [トリガー] 構成案をワークスで共有 + テンプレート確定

【Phase 2: 素材準備】
⑤ 素材撮影（川崎 cc:青柳）
   ▼ [トリガー] 撮影データを指定フォルダに格納

【Phase 3: デザイン】
⑥ デザイン作成（河合 cc:青柳）
   ▼ [トリガー] デザインデータ完成報告

【Phase 4: 実装】
⑦ コーディング（河合 cc:青柳）
   ▼ [トリガー] 実装完了報告

【Phase 5: 確認・納品】
⑧ 企業担当へ確認依頼（河合 cc:青柳）
   ▼ [トリガー] 最終承認 + キックオフ日程確定
⑨ キックオフMTG（河合 cc:青柳）
   ▼ [トリガー] 議事録共有 + 運用開始連絡

【Phase 6: 運用】
⑩ HP内容更新（担当者）※継続運用
⑪ 月次FB（河合 cc:青柳）※毎月
   ▼ [トリガー] FB議事録共有 + ネクスト日程確定
   └─→ Phase 6を繰り返し
```

## 3. 各フェーズ詳細

### Phase 0: 受注・立ち上げ

#### No.0 受注・初回ヒアリング

| 項目 | 内容 |
|------|------|
| **担当** | 営業 → 河合（cc:青柳） |
| **次の業務** | 構成設計・テンプレート選定（河合） |

**業務内容:**

1. **受注確定・ワークス立ち上げ**
   - ワークスで企業グループを新規作成
   - 企業基本情報の共有フォーマットに記入
   - 制作担当者をメンション（河合、cc:青柳）

2. **初回ヒアリング日程調整**
   - 先方へ候補日程を提示
   - 参加者（企業担当・営業・河合 cc:青柳）の予定確認
   - カレンダー登録・ワークス報告

3. **初回ヒアリング実施**
   - HPの目的・ターゲットの確認
   - 企業の強み・差別化ポイントのヒアリング
   - 参考サイトの確認
   - 撮影日程の調整
   - NOTTA等で議事録作成

**GAS機能（ヒアリングシートから）:**
- 📥 企業情報入力（ワークス貼り付け）
- 📢 受注・キックオフ連絡
- 📩 先方へ日程調整・フォーム記入依頼メール

**トリガー:** 議事録のワークスグループ共有

### Phase 1: ヒアリング・企画

#### No.1 構成設計・テンプレート選定

| 項目 | 内容 |
|------|------|
| **担当** | 河合（cc:青柳） |
| **次の業務** | 素材撮影（川崎） |
| **使用ツール** | SANKOU, HPテンプレート |

**業務内容:**

1. **ヒアリング内容の整理**
   - 議事録から要件を抽出
   - ヒアリングシートへ転記

2. **テンプレート選定**
   - 企業の特性に合わせてテンプレートを選択
   - 5種類から最適なものを提案

   | テンプレート | 特徴 | 適した企業 |
   |-------------|------|-----------|
   | ストーリー型 | 社員インタビュー重視 | 人材が魅力の企業 |
   | データ訴求型 | 数字・実績重視 | 成長中企業、実績アピール |
   | ビジュアル型 | 写真・動画重視 | デザイン業界、製造業 |
   | シンプル型 | 1ページ完結 | 小規模企業、急ぎの案件 |
   | Q&A解決型 | 不安解消重視 | 採用課題が明確な企業 |

3. **構成案の作成**
   - ページ構成を決定
   - 各セクションの内容を設計
   - ワイヤーフレーム作成

**GAS機能:**
- 📋 構成案を作成（プロンプト生成）
- 文字起こしからAI用プロンプト生成
- JSONパースしてシートに転記

**トリガー:** 構成案をワークスで共有 + テンプレート確定

### Phase 4: 実装

#### No.4 コーディング

| 項目 | 内容 |
|------|------|
| **担当** | 河合（cc:青柳） |
| **次の業務** | 確認依頼（河合） |
| **使用ツール** | VSCode, Claude Code |
| **成果物** | Webサイト |

**業務内容:**

1. **Claude Codeを起動**
   - VS Codeでプロジェクトを開く
   - WSLで再度開く（左下の緑アイコン → Reopen Folder in WSL）
   - ターミナルを開く（Ctrl + `）
   - `claude` と入力してEnter

2. **作業前の準備**
   - Claude Codeに `git pull してください` と指示
   - 最新の状態に更新してから作業開始

3. **テンプレートのカスタマイズ**
   - 修正マニュアルを参照して指示文をコピー
   - Claude Codeに修正内容を指示
   - 企業固有の情報（テキスト、画像、色）を反映

4. **動作確認**
   - Claude Codeに `npm run dev を実行してください` と指示
   - ブラウザで http://localhost:3000 を確認
   - 表示崩れ、リンク動作、レスポンシブをチェック

5. **コミット＆プッシュ**
   - Claude Codeに `コミットしてプッシュしてください` と指示

6. **デプロイ**
   - Vercelで自動デプロイ（プッシュ時に自動）
   - 本番URLで最終確認

## 5. GASによる自動化

### 5.1 ヒアリングシートの構成

HP制作用のヒアリングシートには以下の機能があります。

**セクション別ヒアリングシート（30分で完了）:**

| セクション | 項目 |
|-----------|------|
| 全体設計 | サイトの目的、ターゲット、与えたい印象、参考サイト |
| ヘッダー | メニュー項目、電話番号表示、CTAボタン |
| ヒーローセクション | キャッチコピー、画像イメージ |
| アバウト | 会社紹介、理念 |
| サービス紹介 | サービス①②③の名称と特徴 |
| 強み | 強み①②③ |
| 実績・お客様の声 | 実績、testimonial |
| 会社情報 | 会社名、代表者、所在地等 |
| ニュース | カテゴリ、更新頻度 |
| CTA・お問い合わせ | CVポイント、電話番号、フォーム項目 |
| フッター | SNS、リンク一覧 |

### 5.2 GASメニュー構成

```
ヒアリング
├── シートを作成
├── 回答をクリア
├── ─────────────
├── 文字起こしからプロンプト生成
├── JSONをパースして入力
├── ─────────────
├── JSONで出力
└── YAMLで出力
```

### 5.3 文字起こしからの自動入力

**使い方:**

1. 初回打ち合わせをNOTTAで録音・文字起こし
2. 「文字起こしからプロンプト生成」を選択
3. 文字起こしを貼り付け
4. 生成されたプロンプトをChatGPT/Claudeに貼り付け
5. AIの出力（JSON）を「JSONをパースして入力」で取り込み
6. 差分を確認してチェックした項目のみ転記

**メリット:**
- 30分の打ち合わせ内容を5分でシートに反映
- 漏れなく情報を抽出
- 担当者の入力負担を軽減

## 6. テンプレート選定ガイド

### 6.1 テンプレート一覧

| ID | 名称 | ページ数 | 特徴 |
|----|------|---------|------|
| story-type | ストーリー型 | 7ページ | 社員インタビュー重視、共感採用 |
| data-driven | データ訴求型 | 7ページ | 数字・実績重視、グラフ活用 |
| visual | ビジュアル型 | 9ページ | 写真・動画重視、没入感 |
| simple | シンプル型 | 1ページ | ミニマルデザイン、1ページ完結 |
| qa-type | Q&A解決型 | 7ページ | 不安解消、FAQ重視 |

### 6.2 選定フローチャート

```
企業の採用課題は何か？
│
├─ 「人」が魅力 → ストーリー型
│   └─ 社員の働き方、想いを伝えたい
│
├─ 実績・データで訴求 → データ訴求型
│   └─ 成長率、離職率、平均年齢等をアピール
│
├─ ビジュアルが強み → ビジュアル型
│   └─ オフィス、製品、作業風景が映える
│
├─ シンプルに伝えたい → シンプル型
│   └─ 予算・納期が限られている
│
└─ 応募者の不安を解消 → Q&A解決型
    └─ 「未経験OK？」「残業は？」等の疑問に答える
```

### 6.3 各テンプレートの詳細

#### ストーリー型

**適した企業:**
- 社員の魅力で勝負する企業
- 若手が活躍している企業
- 社風・カルチャーを重視する企業

**ページ構成:**
- HOME
- 私たちのストーリー
- 先輩社員の声
- 1日の流れ
- 募集職種
- はたらく環境
- 応募する

#### データ訴求型

**適した企業:**
- 成長中の企業
- 数字で訴求できる実績がある企業
- 業界比較で優位性がある企業

**ページ構成:**
- HOME
- 企業データ
- 数字で見る働きやすさ
- キャリアパス
- 募集要項
- よくある質問
- 応募する

#### ビジュアル型

**適した企業:**
- デザイン・クリエイティブ系企業
- 製造業（製品・工場が映える）
- オフィス環境が充実している企業

**ページ構成:**
- HOME
- GALLERY
- PEOPLE
- OFFICE
- CULTURE
- JOBS
- ENTRY

#### シンプル型

**適した企業:**
- スタートアップ・小規模企業
- 予算・納期が限られている
- まずは最低限の情報を出したい

**ページ構成:**
- 1ページに全セクションを集約

#### Q&A解決型

**適した企業:**
- 未経験採用を強化したい企業
- 業界イメージにネガティブ要素がある企業
- 応募前の不安が明確な企業

**ページ構成:**
- HOME（不安TOP5）
- 応募前の疑問
- 選考について
- 入社後について
- 働き方Q&A
- 募集要項
- まだ不安がある方へ

## 9. 品質チェックリスト

### 9.1 デザインチェック

- [ ] 企業のブランドカラーが正しく反映されているか
- [ ] ロゴの表示は適切か（サイズ、位置）
- [ ] フォントは読みやすいか
- [ ] 余白・レイアウトのバランスは適切か
- [ ] 画像の解像度は十分か
- [ ] モバイル表示を考慮しているか

### 9.2 実装チェック

- [ ] 全ページで表示崩れがないか（PC/スマホ）
- [ ] リンクは全て正しく動作するか
- [ ] フォームは正しく送信できるか
- [ ] 画像の読み込みは問題ないか
- [ ] ページ表示速度は許容範囲か
- [ ] SSL証明書は有効か

### 9.3 コンテンツチェック

- [ ] 会社名・担当者名の表記は正しいか
- [ ] 誤字脱字はないか
- [ ] 電話番号・メールアドレスは正しいか
- [ ] 住所・アクセス情報は正しいか
- [ ] 募集要項の内容に相違はないか
- [ ] プライバシーポリシーは適切か

### 9.4 SEOチェック

- [ ] titleタグは適切に設定されているか
- [ ] meta descriptionは設定されているか
- [ ] OGP画像は設定されているか
- [ ] 見出しタグ（h1〜h6）は適切に使われているか
- [ ] 画像のalt属性は設定されているか
- [ ] sitemap.xmlは生成されているか
