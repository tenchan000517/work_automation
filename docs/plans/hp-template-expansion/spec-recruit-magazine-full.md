# Recruit Magazine テンプレート 完全設計仕様書

**作成日**: 2026-02-01
**テンプレート名**: Recruit Magazine
**型（Type）**: Recruit（採用特化）
**表現（Design）**: Magazine（読ませるエディトリアル）

---

## 概要

### コンセプト
求職者が「この会社で働きたい」と思うまでの心理プロセスを、**雑誌を読むように追体験させる**。
情報を羅列するのではなく、**ストーリーとして会社を理解させる**。
社員インタビューを中心に据え、「人」を通じて会社の魅力を伝える。

### 想定クライアント
- 採用に苦戦している中小企業（製造業、建設業、サービス業）
- 人材確保が経営課題の企業
- 「うちの会社の良さが伝わらない」と感じている企業
- 採用サイトを企業サイトと別に持ちたい企業

### 想定ユーザー（閲覧者）
- 高校新卒者（16-18歳）: 不安が大きい、先輩の姿を見たい、厳しすぎないか心配
- 中途採用者（25-40歳）: 前職との比較、成長機会、働きやすさを重視
- 保護者: 子供が働く会社の雰囲気、安定性を確認したい

### 達成すべきこと
1. 応募数の増加（現状比1.5倍以上）
2. 入社後のミスマッチ低減（離職率改善）
3. 「この会社で働きたい」という感情の喚起
4. 会社の雰囲気・人柄が伝わるサイト体験

## PART 1: 全体戦略

### 1-1. このテンプレートの独自価値

#### 専門家①（WEBマーケティング）の視点

**Standard テンプレートとの決定的な違い**:

Standard は「情報を整理して提示する」設計。Recruit Magazine は「感情を動かして行動させる」設計。

具体的な違い:
- **情報提示 → ストーリー導線**: Standard が「会社概要」「事業内容」と情報を並列に置くのに対し、Recruit Magazine は「この会社に入ったらどんな未来があるか」を時系列で追体験させる
- **スペック → パーソナリティ**: 福利厚生の箇条書きではなく、「この制度で実際に助かった」という社員の声を中心に据える
- **閲覧 → 没入**: 離脱率の高い箇条書き形式を排し、雑誌記事のように「読み進めたくなる」構成

**このテンプレートでしか実現できないコンバージョン戦略**:

1. **マイクロコミットメントの連鎖設計**
   - 各ページに「小さなYES」を積み重ねるポイントを設置
   - 例: 「この社員の話、もっと聞きたい？」→ 詳細へ → 「こんな仕事に興味がある？」→ 職種詳細へ
   - 最終的なエントリーまでに5-7回の小さな心理的コミットメントを獲得

2. **ペルソナ別入口の最適化**
   - 高校生向け: 「先輩の1日」から入る導線
   - 中途向け: 「キャリアアップストーリー」から入る導線
   - 保護者向け: 「社長メッセージ」から入る導線
   - 同じサイトでも入口によって体験が変わる

3. **離脱防止の心理トリガー**
   - 平均離脱ポイント（3-4スクロール目）に必ず「人の顔」を配置
   - 人間の顔は無意識に視線を引き付け、離脱を0.3-0.5秒遅らせる
   - その間に次のコンテンツへの興味を喚起

**競合サイトとの差別化**:

| 競合の典型 | Recruit Magazine |
|-----------|------------------|
| 社員の集合写真1枚 | 個別の表情にフォーカスした5人以上の写真群 |
| 「若手活躍中！」の文言 | 入社3年目の具体的な成長ストーリー |
| 福利厚生の箇条書き | 「この制度のおかげで」というエピソード |
| 応募フォームへの直リンク | 心理的準備を整えてからのエントリー誘導 |

#### 専門家②（行動心理学）の視点

**活用する心理的メカニズム**:

1. **社会的証明（Social Proof）の精緻化**
   - 単なる「社員の声」ではなく、**似た境遇の人**の証言を戦略的に配置
   - 高校生が見るページには高校新卒入社の社員を配置
   - 中途が見るページには中途入社でキャリアアップした社員を配置
   - 「自分と同じ状況の人がうまくいっている」という認知が最も強い動機付けになる

2. **物語輸送（Narrative Transportation）**
   - 人は物語に没入すると、批判的思考が低下し、感情的な影響を受けやすくなる
   - Magazine 形式は「読み物」として没入させ、「この会社、いいかも」という感情を自然に生成
   - 論理的説得（給与、福利厚生）だけでは動かない層を動かす

3. **単純接触効果の最大化**
   - 同じ社員を複数ページで異なる文脈で登場させる
   - TOP で笑顔 → People で深いインタビュー → Culture で働く姿
   - 「この人のことを知っている」という親近感が、会社全体への親近感に転化

4. **損失回避バイアスの活用**
   - 「応募しないと失うもの」を暗示的に示す
   - 「今年の新卒は◯名採用。残り◯名」のようなリアルタイム表示
   - 「〇〇さんはこの会社を選んで、今は〜」という機会獲得ストーリー

**ユーザーの意思決定プロセスへの作用点**:

```
認知 → 興味 → 欲求 → 比較 → 確信 → 行動
        ↑      ↑             ↑
     People   Work/Culture   FAQ/Entry
     の写真   の具体性       の安心感
```

**Standard では達成できない心理効果**:

- **感情記憶の形成**: 「あの社員の笑顔」という感情的記憶が残る。情報は忘れても感情は残る
- **疑似体験による確信**: 「1日の流れ」を追体験することで「自分でもできそう」という自己効力感が生まれる
- **集団帰属欲求の喚起**: 「この人たちの仲間になりたい」という欲求を、情報ではなく雰囲気で生成

#### 専門家③（WEBデザイン）の視点

**ビジュアルアイデンティティ**:

**キーワード**: 「余白で語る、写真で動かす、文字で刻む」

従来の採用サイトが「情報を詰め込む」設計だったのに対し、Recruit Magazine は「余白を恐れない」ことでプレミアム感と読みやすさを両立。

**ビジュアルの3原則**:
1. **写真は「語る」写真のみ**: 作り笑顔の集合写真ではなく、仕事中の自然な表情、手元のアップ、横顔など「物語が見える」写真
2. **文字は「見せる」と「読ませる」を分離**: 大見出しは視認性、本文は可読性に特化
3. **余白は「意図」を持つ**: 何も置かない空間にも「ここで一息」「ここで切り替え」という意味がある

**「Magazine」表現の具体化**:

| 雑誌の特徴 | WEB での実装 |
|-----------|-------------|
| 見開きページ | 横幅いっぱいのセクション切り替え |
| 写真と文字の重なり | 画像の上にオーバーレイテキスト |
| 引用の囲み | 大きな引用符と字下げ |
| ページめくり | スクロールに応じたパララックス |
| 章扉 | セクション冒頭の大きな余白とタイトル |
| 編集後記 | 採用担当者の個人的なメッセージ |

**AIデザインとの差別化ポイント**:

AIが生成しがちなパターン:
- ❌ 全要素に均一な角丸カード
- ❌ 紫〜青のグラデーション背景
- ❌ 全セクションで同じ余白（80px など）
- ❌ すべての見出しをセンタリング
- ❌ アイコンとテキストの1:1対応

Recruit Magazine の対抗策:
- ✅ カード使用は限定的。使う場合も角丸なし、影なしのフラット設計
- ✅ 白/グレー/ブランドカラーのソリッドな色使い
- ✅ 余白は40px〜160pxまで意図的にバラつかせる
- ✅ 見出しは左寄せ基本。センタリングは「特別な瞬間」のみ
- ✅ アイコンは補助。写真と文字が主役

### 1-2. カラー戦略

**メインカラーの推奨系統と心理効果**:

採用サイトにおいて「業種」よりも「伝えたい印象」でカラーを決定する。

| 伝えたい印象 | 推奨色 | HSL値例 | 心理効果 |
|-------------|-------|--------|---------|
| 信頼・安定 | ネイビーブルー | hsl(220, 60%, 25%) | 「ちゃんとした会社」という安心感。製造業・建設業向け |
| 成長・活力 | ディープグリーン | hsl(160, 50%, 30%) | 「伸びている会社」という期待感。IT・サービス業向け |
| 情熱・挑戦 | ブリックレッド | hsl(10, 70%, 40%) | 「熱い会社」という共感。ベンチャー・飲食業向け |
| 温かさ・人柄 | テラコッタ | hsl(25, 50%, 45%) | 「人を大切にする会社」という親近感。介護・教育向け |

**重要**: 彩度を抑える（50-70%）。彩度100%のビビッドカラーは「安っぽい」「煽っている」印象を与える。

**アクセントカラーの使いどころ**:

アクセントカラーはメインカラーの補色ではなく、**同系色のバリエーション**を使用。

使用箇所（優先順位順）:
1. **CTAボタン**: メインカラーより10%明るく、hover時に元の色に戻る
2. **リンクテキスト**: メインカラーと同一
3. **数字のハイライト**: 「入社3年目」「年間休日120日」などの数字に適用
4. **セクション区切り線**: 極細（1px）で使用

**背景色のバリエーションとリズム**:

```
セクション1: 白（#FFFFFF）
セクション2: 極薄グレー（#FAFAFA）
セクション3: 白（#FFFFFF）
セクション4: メインカラー10%透過（rgba(主色, 0.08)）
セクション5: 白（#FFFFFF）
```

**法則**: 3セクション以上連続で同じ背景色を使わない。ただし白は2連続まで許容。

**避けるべき色の組み合わせ**:

- ❌ 紫系グラデーション（AIデザインの典型）
- ❌ 蛍光イエロー × 黒（警告色。採用には不適切）
- ❌ パステルピンク × パステルブルー（幼稚な印象）
- ❌ 赤 × 緑（色覚多様性への配慮不足）
- ❌ 白背景 × グレー（#999以上）テキスト（コントラスト比不足）

### 1-3. タイポグラフィ戦略

**見出しのサイズ階層**:

```css
/* PC */
H1: 48px / line-height: 1.3 / font-weight: 700
   → ページタイトル、ヒーローのメインコピー

H2: 32px / line-height: 1.4 / font-weight: 700
   → セクションタイトル

H3: 24px / line-height: 1.5 / font-weight: 600
   → サブセクション、カード内タイトル

H4: 18px / line-height: 1.6 / font-weight: 600
   → 小見出し、FAQ質問文

/* SP */
H1: 32px
H2: 24px
H3: 20px
H4: 16px
```

**本文とのコントラスト**:

```css
本文: 16px / line-height: 1.9 / font-weight: 400 / color: #333
小文字: 14px / line-height: 1.7 / color: #666
キャプション: 12px / line-height: 1.5 / color: #888
```

**重要**: 本文の line-height は1.9を死守。日本語は1.5では詰まりすぎて読みにくい。

**日本語特有の考慮点**:

1. **letter-spacing**:
   - 見出し: 0.05em（やや広め）
   - 本文: 0（デフォルト）
   - 数字を含む場合: 数字のみ font-feature-settings: "tnum" で等幅

2. **フォントファミリー**:
   ```css
   font-family: "Noto Sans JP", "Hiragino Kaku Gothic ProN", "メイリオ", sans-serif;
   ```
   - 游ゴシックは使用しない（Windows で細すぎる問題）

3. **改行位置**:
   - word-break: break-all は禁止（英単語が壊れる）
   - overflow-wrap: break-word を使用

**「Magazine」表現におけるタイポグラフィの役割**:

| 雑誌的表現 | 実装方法 |
|-----------|---------|
| 大見出しのインパクト | H1を48pxで使用。1行に収める |
| 引用文の強調 | 24px + イタリック風（游明朝）+ 左に縦線 |
| リード文 | 18px + color:#555 + 本文より0.1em広いletter-spacing |
| 編集ノート風 | 手書き風フォント使用可（Yuji Syuku等）+ 斜め配置 |

### 1-4. 余白とリズム戦略

**セクション間の余白の考え方**:

```
余白の基本単位: 20px

小余白:  40px (2単位) → 同一トピック内の区切り
中余白:  80px (4単位) → トピック切り替え
大余白: 120px (6単位) → 章の切り替え、ページの区切り感
特大:   160px (8単位) → 「ここで一息」という意図的な間
```

**セクションごとの余白割り当て**:

| セクション | 上余白 | 下余白 | 理由 |
|-----------|-------|-------|------|
| ヒーロー後 | - | 80px | 本文への導入として標準 |
| 数字セクション | 120px | 80px | 前との断絶を強調し、数字に注目させる |
| 人物インタビュー | 80px | 120px | 読後感を持たせ、次への期待を作る |
| CTA前 | 160px | - | 意思決定の「間」を作る |

**「Magazine」表現における余白の意味**:

雑誌は「何も書かない空間」で高級感を演出する。WEBでも同様。

- **贅沢な余白** = 「この会社には余裕がある」という無言のメッセージ
- **写真の周囲の余白** = 「この人を大切にしている」という表現
- **CTA前の余白** = 「急かさない」という姿勢の表明

**密と疎のリズム設計**:

```
TOP:    疎 → 密 → 疎 → 密 → 疎
        ヒーロー(疎) → 社員紹介3名(密) → 数字(疎) → ニュース(密) → CTA(疎)

People: 疎 → 密 → 疎 → 密 → 疎
        導入(疎) → 人物1詳細(密) → 余白(疎) → 人物2詳細(密) → ...
```

**法則**: 密→密、疎→疎は禁止。必ず交互に配置。

**均一な余白を避ける具体的な方法**:

1. **上下の非対称**: 上余白と下余白は必ず異なる値にする（例: 上120px、下80px）
2. **セクション内の余白変化**: 同一セクション内でも要素間で20px/40pxを使い分ける
3. **視覚的重心の調整**: テキストが多いセクションは上余白を大きく、画像が多いセクションは下余白を大きく

### 1-5. アニメーション戦略

**どこにアニメーションを使い、どこに使わないか**:

| 使う | 使わない |
|-----|---------|
| ファーストビュー | ヘッダー/ナビ |
| スクロール最初のフックポイント | すべての要素の出現 |
| CTA周辺 | 本文テキスト |
| 人物の登場 | 背景色の変化 |
| ホバーアクション | ローディング以外の無限ループ |

**「Magazine」表現におけるアニメーションの役割**:

雑誌をめくる動作の代替として、**ページ遷移**と**スクロール**にのみ意味のあるアニメーションを付与。

- ページめくり感 = セクション切り替え時の緩やかなフェード
- 視線誘導 = 重要要素への微細なモーション
- 読み進める楽しさ = スクロールに応じた段階的出現

**過剰にならない基準**:

1. **同時に動く要素は2つまで**
2. **アニメーション時間は0.3-0.6秒**（0.2秒未満は知覚できない、1秒超は待たされる感覚）
3. **easing は ease-out 系統**（加速してから減速は不自然）
4. **transform のみ使用**（top/left のアニメーションは避ける）

**具体的なアニメーションパターン（3種類）**:

**Pattern 1: Fade Up（基本の出現）**
```tsx
// 用途: セクションタイトル、人物写真の出現
const fadeUp = {
  initial: { opacity: 0, y: 30 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.5, ease: [0.25, 0.1, 0.25, 1] }
}
```

**Pattern 2: Stagger Reveal（連続出現）**
```tsx
// 用途: 3つの数字、社員カード群
const container = {
  animate: { transition: { staggerChildren: 0.1 } }
}
const item = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0, transition: { duration: 0.4 } }
}
```

**Pattern 3: Parallax Subtle（微細な視差）**
```tsx
// 用途: 背景写真、装飾要素
// スクロール量の10%だけずれて動く
const parallax = {
  style: {
    transform: `translateY(${scrollY * 0.1}px)`
  }
}
```

**禁止パターン**:
- ❌ Bounce（跳ねる）: 子供っぽい
- ❌ Rotate（回転）: 意味がない動き
- ❌ Scale > 1.1（拡大）: 暴力的な印象
- ❌ 無限ループ: 注意を奪い続ける

### TOPページ（/）

**このページの目的**:
- **専門家①（マーケティング）**: 3秒以内に「採用サイトであること」「どんな会社か」「次に何をすべきか」を伝え、直帰率を40%以下に抑える
- **専門家②（心理学）**: 「この会社、良さそう」という第一印象を形成し、社会的証明（社員の顔）で信頼を獲得
- **専門家③（デザイン）**: サイト全体のトーンを確立し、「Magazine」のような上質な閲覧体験の予感を与える

**ユーザーの状態**:
- **来る前**: 「どんな会社だろう」という漠然とした興味、または求人媒体からの流入で情報収集モード
- **感じてほしいこと**: 「雰囲気が良さそう」「社員が楽しそう」「もっと知りたい」
- **離れる時**: 具体的なページ（People/Work）への遷移意欲、または会社名の記憶定着

#### セクション1: ヒーローエリア

**なぜこのセクションが必要か**:
- **専門家①**: ファーストビューは離脱/継続の判断が0.05秒で行われる。「採用」「人」「メッセージ」を瞬時に伝える必要がある
- **専門家②**: 人間の顔は本能的に注目を集める。笑顔は信頼と親近感を生む。感情的な第一印象がその後の情報処理に影響
- **専門家③**: 「Magazine の表紙」として機能。視覚的インパクトで「読み進めたい」という欲求を喚起

**レイアウト**:
- 構成: 画面高さ90vh。左側30%にテキストブロック（上下中央揃え）、右側70%に社員写真（斜め15度でクロップ、画面外まではみ出し）
- なぜこのレイアウトか: 視線は左上から右下へ流れる。テキストを先に読ませ、写真で感情を動かす。斜めのクロップは「動き」「活力」を表現
- 代替案を採用しない理由:
  - 全面写真+中央テキスト → テキストが読みにくい、どこを見ればいいか迷う
  - グリッド分割 → 静的すぎる、Magazine感がない
  - 動画背景 → 読み込み遅延、モバイルで機能しない

**コンテンツ要素**:
- **メインコピー（H1）**: 1行20文字以内。「仲間と、未来を、つくる。」のような感情に訴える短文。会社名や業種は入れない
- **サブコピー**: 2行60文字以内。「〇〇株式会社は、[業種]を通じて[価値]を届けています。」
- **CTAボタン**: 「社員を知る」（People ページへ）。あえて「応募する」にしない。まだ早い
- **スクロールインジケーター**: 画面下部中央に「↓ Scroll」のアニメーション

**デザイン詳細**:
- 背景: 白（#FFFFFF）。写真を際立たせるため
- 余白: 左テキストブロックはビューポート左端から80px
- 特徴的な表現: 写真の斜めクロップ線はメインカラーの細い線（2px）で縁取り

**アニメーション**:
- 使用する
- メインコピー: 0.3秒後に fade up（duration: 0.5s）
- サブコピー: 0.5秒後に fade up（duration: 0.5s）
- 写真: 0.2秒後に右からスライドイン（translateX: 100 → 0, duration: 0.7s）
- スクロールインジケーター: 1.5秒後に出現、上下に2pxの揺れを無限ループ（これのみループ許可）

**SP（スマートフォン）での変化**:
- レイアウト変更: 縦積み。写真が上60vh、テキストが下40vh
- サイズ変更: H1は32px、写真は斜めクロップなし（シンプルな矩形）
- 省略要素: なし

**このセクションでの心理効果**:
- ユーザーが感じること: 「活気がある会社だな」「社員が楽しそう」
- 次のセクションへの期待: 「もっとこの会社のことを知りたい」

#### セクション2: コンセプトメッセージ

**なぜこのセクションが必要か**:
- **専門家①**: ヒーローで感情を掴んだ後、「論理」で補強する。なぜこの会社が存在するのかを簡潔に
- **専門家②**: 人は感情で決断し、論理で正当化する。感情の後に理由を提示することで「この会社を選ぶ理由」を提供
- **専門家③**: ヒーローの視覚的密度の後に、余白の多い「読ませる」セクションを配置してリズムを作る

**レイアウト**:
- 構成: 左右マージン20%の中央寄せテキストブロック。背景なし、装飾最小限
- なぜこのレイアウトか: 雑誌の巻頭言のように「読むことに集中」させる。横幅を狭めることで1行あたりの文字数を適正（30-40文字）に保つ

**コンテンツ要素**:
- **セクションラベル**: 「CONCEPT」（英字、letter-spacing: 0.3em、12px、色: #999）
- **見出し（H2）**: 20文字以内。「〇〇で、日本の△△を支える」
- **本文**: 150-200文字。3段落に分け、1段落50-70文字

**デザイン詳細**:
- 背景: 白（#FFFFFF）
- 余白: 上120px、下80px。上を大きくすることで「新しい章の始まり」を示す
- 特徴的な表現: 見出しの上に細い横線（幅60px、高さ1px、メインカラー）

**アニメーション**:
- 使用する
- テキストブロック全体: スクロールで画面に入った時に fade up（duration: 0.5s）

**SP での変化**:
- レイアウト変更: 左右マージン20px（パーセントではなく固定値）
- サイズ変更: H2は24px

**このセクションでの心理効果**:
- ユーザーが感じること: 「この会社、ちゃんと考えて事業やってるんだな」
- 次のセクションへの期待: 「具体的にどんな人が働いてるんだろう」

```typescript
interface NumberBlockProps {
  number: number | string;
  unit?: string;
  label: string;
  size: 'large' | 'medium' | 'small';
  animate?: boolean;
}
```

**デザイン仕様**:
- large: 数字72px、ブロック幅2/3
- medium: 数字56px、ブロック幅1/2
- small: 数字48px、ブロック幅1/3

### 4. TimelineHorizontal

**用途**: Work「1日の流れ」、Culture「研修スケジュール」
**なぜ必要か**: 横向きタイムラインは「1日」「プロセス」を表現

```typescript
interface TimelineHorizontalProps {
  title?: string;
  items: {
    time: string;
    label: string;
    description?: string;
    thumbnail?: string;
  }[];
}
```

**デザイン仕様**:
- 全幅、高さ200px程度
- SPで縦向きに自動変換

### 5. QuoteBlock

**用途**: People 引用強調、Culture 社員の声
**なぜ必要か**: 雑誌のプルクオート再現

```typescript
interface QuoteBlockProps {
  quote: string;
  author?: { name: string; position?: string; photo?: string };
  size?: 'large' | 'medium';
  align?: 'left' | 'center';
}
```

**デザイン仕様**:
- large: 24px（インタビュー内）
- medium: 18px（社員の声）
- 左に縦線（4px、メインカラー）

### 6. PhotoGallery

**用途**: Culture オフィスギャラリー
**なぜ必要か**: 不規則グリッド、ライトボックス機能

```typescript
interface PhotoGalleryProps {
  photos: { src: string; alt: string; size: 'large' | 'medium' | 'small' }[];
  columns?: number;
}
```

**デザイン仕様**:
- ガター8px
- ホバー時scale: 1.02

### 7. BenefitHighlight

```typescript
  };
```

```typescript
interface RecruitmentCardProps {

}
```
