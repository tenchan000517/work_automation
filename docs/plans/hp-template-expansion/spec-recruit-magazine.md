# Recruit Magazine テンプレート 完全設計仕様書

**作成日**: 2026-02-01
**テンプレート名**: Recruit Magazine
**型（Type）**: Recruit（採用特化）
**表現（Design）**: Magazine（読ませるエディトリアル）

---

## 概要

### コンセプト
求職者が「この会社で働きたい」と思うまでの心理プロセスを、**雑誌を読むように追体験させる**。
情報を羅列するのではなく、**ストーリーとして会社を理解させる**。
社員インタビューを中心に据え、「人」を通じて会社の魅力を伝える。

### 想定クライアント
- 採用に苦戦している中小企業（製造業、建設業、サービス業）
- 人材確保が経営課題の企業
- 「うちの会社の良さが伝わらない」と感じている企業
- 採用サイトを企業サイトと別に持ちたい企業

### 想定ユーザー（閲覧者）
- 高校新卒者（16-18歳）: 不安が大きい、先輩の姿を見たい、厳しすぎないか心配
- 中途採用者（25-40歳）: 前職との比較、成長機会、働きやすさを重視
- 保護者: 子供が働く会社の雰囲気、安定性を確認したい

### 達成すべきこと
1. 応募数の増加（現状比1.5倍以上）
2. 入社後のミスマッチ低減（離職率改善）
3. 「この会社で働きたい」という感情の喚起
4. 会社の雰囲気・人柄が伝わるサイト体験

## PART 1: 全体戦略

### 1-1. このテンプレートの独自価値

#### 専門家①（WEBマーケティング）の視点

**Standard テンプレートとの決定的な違い**:

Standard は「情報を整理して提示する」設計。Recruit Magazine は「感情を動かして行動させる」設計。

具体的な違い:
- **情報提示 → ストーリー導線**: Standard が「会社概要」「事業内容」と情報を並列に置くのに対し、Recruit Magazine は「この会社に入ったらどんな未来があるか」を時系列で追体験させる
- **スペック → パーソナリティ**: 福利厚生の箇条書きではなく、「この制度で実際に助かった」という社員の声を中心に据える
- **閲覧 → 没入**: 離脱率の高い箇条書き形式を排し、雑誌記事のように「読み進めたくなる」構成

**このテンプレートでしか実現できないコンバージョン戦略**:

1. **マイクロコミットメントの連鎖設計**
   - 各ページに「小さなYES」を積み重ねるポイントを設置
   - 例: 「この社員の話、もっと聞きたい？」→ 詳細へ → 「こんな仕事に興味がある？」→ 職種詳細へ
   - 最終的なエントリーまでに5-7回の小さな心理的コミットメントを獲得

2. **ペルソナ別入口の最適化**
   - 高校生向け: 「先輩の1日」から入る導線
   - 中途向け: 「キャリアアップストーリー」から入る導線
   - 保護者向け: 「社長メッセージ」から入る導線

3. **離脱防止の心理トリガー**
   - 平均離脱ポイント（3-4スクロール目）に必ず「人の顔」を配置
   - 人間の顔は無意識に視線を引き付け、離脱を0.3-0.5秒遅らせる

#### 専門家②（行動心理学）の視点

**活用する心理的メカニズム**:

1. **社会的証明（Social Proof）の精緻化**
   - 「自分と同じ状況の人がうまくいっている」という認知が最も強い動機付けになる

2. **物語輸送（Narrative Transportation）**
   - 人は物語に没入すると、批判的思考が低下し、感情的な影響を受けやすくなる

3. **単純接触効果の最大化**
   - 同じ社員を複数ページで異なる文脈で登場させる

4. **損失回避バイアスの活用**
   - 「応募しないと失うもの」を暗示的に示す

#### 専門家③（WEBデザイン）の視点

**ビジュアルアイデンティティ**:

**キーワード**: 「余白で語る、写真で動かす、文字で刻む」

**ビジュアルの3原則**:
1. **写真は「語る」写真のみ**: 作り笑顔の集合写真ではなく、仕事中の自然な表情
2. **文字は「見せる」と「読ませる」を分離**: 大見出しは視認性、本文は可読性に特化
3. **余白は「意図」を持つ**: 何も置かない空間にも意味がある

**AIデザインとの差別化ポイント**:
- ✅ カード使用は限定的。角丸なし、影なしのフラット設計
- ✅ 白/グレー/ブランドカラーのソリッドな色使い
- ✅ 余白は40px〜160pxまで意図的にバラつかせる
- ✅ 見出しは左寄せ基本。センタリングは「特別な瞬間」のみ

### 1-2. カラー戦略

**メインカラーの推奨系統と心理効果**:

| 伝えたい印象 | 推奨色 | HSL値例 | 心理効果 |
|-------------|-------|--------|---------|
| 信頼・安定 | ネイビーブルー | hsl(220, 60%, 25%) | 製造業・建設業向け |
| 成長・活力 | ディープグリーン | hsl(160, 50%, 30%) | IT・サービス業向け |
| 情熱・挑戦 | ブリックレッド | hsl(10, 70%, 40%) | ベンチャー・飲食業向け |
| 温かさ・人柄 | テラコッタ | hsl(25, 50%, 45%) | 介護・教育向け |

**重要**: 彩度を抑える（50-70%）。彩度100%のビビッドカラーは「安っぽい」印象を与える。

**背景色のバリエーションとリズム**:
```
セクション1: 白（#FFFFFF）
セクション2: 極薄グレー（#FAFAFA）
セクション3: 白（#FFFFFF）
セクション4: メインカラー10%透過
セクション5: 白（#FFFFFF）
```

**避けるべき色の組み合わせ**:
- ❌ 紫系グラデーション（AIデザインの典型）
- ❌ 蛍光イエロー × 黒
- ❌ パステルピンク × パステルブルー
- ❌ 白背景 × グレー（#999以上）テキスト

### 1-4. 余白とリズム戦略

**セクション間の余白**:
```
小余白:  40px → 同一トピック内の区切り
中余白:  80px → トピック切り替え
大余白: 120px → 章の切り替え
特大:   160px → 「ここで一息」という意図的な間
```

**密と疎のリズム**:
```
TOP: 疎 → 密 → 疎 → 密 → 疎
     ヒーロー → 社員紹介3名 → 数字 → ニュース → CTA
```

**法則**: 密→密、疎→疎は禁止。必ず交互に配置。

### 1-5. アニメーション戦略

**使う場所**:
- ファーストビュー
- スクロール最初のフックポイント
- CTA周辺
- 人物の登場
- ホバーアクション

**使わない場所**:
- ヘッダー/ナビ
- 本文テキスト
- 背景色の変化
- ローディング以外の無限ループ

**具体的なパターン（3種類）**:

```tsx
// Pattern 1: Fade Up（基本の出現）
const fadeUp = {
  initial: { opacity: 0, y: 30 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.5, ease: [0.25, 0.1, 0.25, 1] }
}

// Pattern 2: Stagger Reveal（連続出現）
const container = {
  animate: { transition: { staggerChildren: 0.1 } }
}
const item = {
  initial: { opacity: 0, y: 20 },
  animate: { opacity: 1, y: 0, transition: { duration: 0.4 } }
}

// Pattern 3: Parallax Subtle（微細な視差）
const parallax = {
  style: { transform: `translateY(${scrollY * 0.1}px)` }
}
```

**禁止パターン**:
- ❌ Bounce / Rotate / Scale > 1.1 / 無限ループ

### TOPページ（/）

**目的**:
- 3秒以内に「採用サイトであること」「どんな会社か」「次に何をすべきか」を伝える
- 直帰率を40%以下に抑える

#### セクション1: ヒーローエリア

**レイアウト**:
- 画面高さ90vh
- 左30%にテキストブロック（上下中央揃え）
- 右70%に社員写真（斜め15度でクロップ、画面外まではみ出し）

**コンテンツ要素**:
- メインコピー（H1）: 1行20文字以内。感情に訴える短文
- サブコピー: 2行60文字以内
- CTAボタン: 「社員を知る」（People ページへ）
- スクロールインジケーター: 画面下部中央

**デザイン詳細**:
- 背景: 白（#FFFFFF）
- 余白: 左テキストブロックはビューポート左端から80px
- 写真の斜めクロップ線はメインカラーの細い線（2px）で縁取り

**アニメーション**:
- メインコピー: 0.3秒後に fade up（duration: 0.5s）
- サブコピー: 0.5秒後に fade up（duration: 0.5s）
- 写真: 0.2秒後に右からスライドイン

**SP**: 縦積み。写真が上60vh、テキストが下40vh

#### セクション2: コンセプトメッセージ

**レイアウト**: 左右マージン20%の中央寄せテキストブロック

**コンテンツ要素**:
- セクションラベル: 「CONCEPT」（12px、#999）
- 見出し（H2）: 20文字以内
- 本文: 150-200文字、3段落

**デザイン詳細**:
- 背景: 白
- 余白: 上120px、下80px
- 見出しの上に細い横線（幅60px、高さ1px、メインカラー）

#### セクション3: 社員ピックアップ（3名）

**レイアウト**: 横3カラム、各カラムに人物写真（上2/3）+ テキスト（下1/3）

**コンテンツ要素**:
- 人物写真: 業務中の自然な表情（比率2:3の縦長）
- 名前: 苗字のみ + さん
- 肩書き: 「入社◯年目 / 部署名」
- 一言コメント: 40文字以内
- 詳細リンク: 「インタビューを読む →」

**デザイン詳細**:
- 背景: 極薄グレー（#FAFAFA）
- 余白: 上80px、下100px
- 番号表示（01, 02, 03）: メインカラー20%透過、80px

**アニメーション**: stagger（0.15秒間隔）

#### セクション4: 数字で見る（Company Facts）

**レイアウト**: 不規則なグリッド（大きいブロックと小さいブロックを混在）

**コンテンツ要素（5-6個）**:
- 創業年数、従業員数、平均年齢、男女比、有給取得率、定着率など
- 数字: 48-72px、ラベル: 14px

**デザイン詳細**:
- 背景: メインカラー8%透過
- 余白: 上120px、下80px

**アニメーション**: 数字がカウントアップ（duration: 1.5s）

#### セクション5: 仕事を知る（Work へのリード）

**レイアウト**: 左50%に業務風景写真、右50%にテキスト

**コンテンツ要素**:
- 写真: 手元・作業風景
- リード文 + 職種リスト（3つ）
- CTAリンク: 「仕事の詳細を見る →」

**デザイン詳細**:
- 背景: 白
- 余白: 上80px、下80px

#### セクション6: CTA

**レイアウト**: 中央揃え、2ボタン横並び

**コンテンツ要素**:
- メッセージ: 「あなたの一歩を、待っています。」
- ボタン左: 「募集要項を見る」（アウトライン）
- ボタン右: 「エントリーする」（塗りつぶし）
- 補足: 「まずは話を聞くだけでも歓迎です」

**デザイン詳細**:
- 背景: 極薄グレー
- 余白: 上160px、下120px
- ボタン: 角丸なし
