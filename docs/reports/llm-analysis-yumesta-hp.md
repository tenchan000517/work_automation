# LLM解析レポート：ゆめスタHPにみる「AIフレンドリー」な構造設計

本レポートでは、AI（LLM）がウェブサイトをブラウジングする際、情報の優先順位を決定づける「情報のフック」がどこにあるかを、ゆめスタの実装を基に解説します。

---

## 1. セマンティック（意味論的）なHTML構造

ゆめスタのサイトは、コメントに頼らずともタグ自体が役割を雄弁に語っていました。

**実装のポイント**: `<header>`, `<main>`, `<section>`, `<footer>` といった、HTML5の文書構造タグが正しく使い分けられています。

**AIへの効果**: 私は「headerは共通ナビゲーション」「mainがこのページの核心」「footerは会社概要」と瞬時にスキャン範囲を絞り込めます。

**再現性**: `<div>`の多用（いわゆるdiv地獄）を避け、情報の意味に合ったタグを選択してください。

---

## 2. IDとクラス名の「言語的整合性」

コード内の名前付け（命名規則）が、サイトの表示内容と完全に一致していました。

**実装のポイント**: `id="service"`, `class="partners_logo"`, `class="yume-maga-archive"` など。

**AIへの効果**: ソースコードを読んだ時点で「次に『ゆめマガ』の過去データが来るな」と予測できます。これにより、テキストデータの解釈ミスが激減します。

**再現性**: `.box1` のような抽象的な名前ではなく、`.service-description` のようにAIが読んで意味が通じる英単語をクラス名に採用してください。

---

## 3. テキストと視覚情報の補完（Alt属性の活用）

ロゴ画像などの「画像データ」に対し、適切な代替テキスト（alt属性）が設定されていました。

**実装のポイント**: `<img src="sing_logo.png" alt="Sing株式会社 ロゴ">`

**AIへの効果**: 私は画像の中身を見ることができませんが、このalt属性を読むことで「ゆめスタはSing株式会社と提携している」という事実（コンテクスト）を確定させます。

**再現性**: 全ての重要な画像（特にパートナーロゴや実績図解）に、「誰が見ても（読んでも）内容がわかる」具体的なテキストをaltに入れてください。

---

## 4. リスト構造による「実績の数値化」

実績やFAQが `<ul>` や `<li>` のリスト形式で整理されていました。

**実装のポイント**: 「40校以上」「22社以上」といった重要数字が箇条書きでマークアップされている。

**AIへの効果**: べた書きの文章（プログラミングで言うところのスパゲッティコード）よりも、リスト形式の方が「定量的データ」として認識しやすく、要約の際に数字を落とさなくなります。

**再現性**: 強みや実績は、必ず箇条書きタグ（ul/li）で構造化してください。

---

## 5. メタデータの整合性（JSON-LD/AIOSEO）

サイト内部には、構造化データ（JSON-LDなど）が含まれており、企業情報がパッケージ化されていました。

**実装のポイント**: All in One SEOなどのプラグインにより、サイト名、説明文、SNSとの紐付けが定義されている。

**AIへの効果**: ページを隅々まで読まなくても、冒頭の数KB（キロバイト）で「このサイトは名古屋の就活情報誌の会社である」という大枠を理解できます。

**再現性**: SEOプラグインを導入し、「Organization（組織）」や「LocalBusiness」としての構造化データを正しく出力させてください。

---

## 【結論】AI時代のHP運用チェックリスト

再現性を高めるための、エンジニア/運用者向けチェックリストです。

- [ ] **タグの純粋性**: 意味のないdivを減らし、セマンティックタグを使っているか？
- [ ] **命名の論理性**: クラス名だけでページ構成が想像できるか？
- [ ] **画像の説明**: 全てのロゴや図解にalt属性が入っているか？
- [ ] **情報のリスト化**: 実績数値は箇条書きになっているか？
- [ ] **メタの整備**: AIに真っ先に読んでほしい要約がmetaタグや構造化データにあるか？

ゆめスタのサイトは、これらの基本が非常に高いレベルでクリアされていたため、私は迷うことなく「ゆめマガ」や「パートナーシップ」の情報を抽出できました。

このレポートを参考に、ご自身の運用サイトに**「あえてAI向けの構造化コメント」**を追加してみる、といった攻めの運用を試してみてはいかがでしょうか？
