# 修正・根拠作成マニュアル

## 概要
HPの修正・リデザインを行い、MVPを先方に渡す際のトークスクリプト（デザインの根拠）を作成する。

## 担当者
- 河合

## 使用ツール
- Claude Code
- AI

## フロー

### 1. リデザイン
必要に応じてデザイン調整を行う。
- 素材の配置調整
- 色味の調整
- レイアウトの微調整

### 2. 根拠作成
AIでデザインの意図を説明するトークスクリプトを作成する。

**トークスクリプトの内容:**
- なぜこのデザインにしたか
- ターゲット層に対する訴求ポイント
- 改善提案の根拠

### 3. 最終確認
MVP完成を確認する。
- 全ページの表示確認
- トークスクリプトの内容確認

### 4. ステータス更新
ステータスを更新して次の工程へ。

## チェックポイント
- MVPとして先方に見せられる状態か
- デザインの根拠が説明できるか

## 次の担当者
- 河合（MVP確認・修正）
