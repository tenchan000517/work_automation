## Phase 2A: 原稿作成

### No.4 原稿執筆

| 項目 | 内容 |
|------|------|
| **担当** | 河合, 中尾文香 |
| **次の業務** | 企業担当へ確認依頼（河合） |

#### どんな作業がありますか？

**基本方針**
- 中小企業が大企業に採用で勝つための「惹きつけ型・共感型」求人原稿を作成
- 条件比較ではなく「ここで働く自分が想像できる原稿」を目指す

**原稿構成ルール（この順番で書く）**
1. 求職者の不安・先入観
2. 仕事の意味・価値
3. 作業内容（具体）
4. 支え方・育て方
5. 会社の考え方
6. 条件・制度（最後）

**作業手順**
1. ヒアリングシートを基にライティング
2. **Pairsona**: 編集データを指定フォルダからダウンロード → YouTubeアップロード → リンク生成 → 原稿内に挿入
3. **エンゲージ**: フォルダ内から動画ダウンロード → エンゲージ内に挿入
4. ライティング後、原稿のリンクかスクショを各企業のワークスへ共有

**最終チェック**
- □ 他社名に差し替えて成立しないか
- □ 一日の過ごし方が想像できるか
- □ ネガティブを隠していないか
- □ 応募者を無言で絞っていないか

#### 何をしたら次にパスできますか？（トリガー）
**原稿のリンクまたはスクショを各企業ワークスへ共有**

### No.3 撮影

| 項目 | 内容 |
|------|------|
| **担当** | 川崎 |
| **次の業務** | 編集（河合） |

#### どんな作業がありますか？

**撮影の大前提**
- 縦動画（9:16）で撮影
- HD・60FPSで撮影（こだわりたい撮影は4K・60FPSでも可）
- 寄りすぎない（編集で「寄る」ことはできるが「引く」ことはできない）
- 動画全体の流れ（導入〜オチ）を意識して撮る

**画角の基本**
- 真正面からは避ける
- 少し斜め（約20〜30度）から撮る
- 被写体の周囲に余白を残す

**寄り撮影時のルール（必須：2パターン撮影）**
1. 定点撮影: カメラ固定、5秒以上
2. 流し撮り: 左→右 または 右→左、ゆっくり一定速度

**撮影後チェック**
- □ 寄りすぎていないか
- □ カメラをゆっくり動かせているか
- □ 定点＋流しの両方を撮れているか
- □ 不要な人・物が映っていないか
- □ 導入とオチが撮れているか

**データ納品**
1. 共有ドライブの指定フォルダを開く
2. 指定のファイル名で格納

**ファイル命名規則**
```
[企業名]_[撮影日]_[連番].mp4
例: 中京商運_20251226_01.mp4
```

#### 何をしたら次にパスできますか？（トリガー）
**撮影データを共有ドライブの指定フォルダに指定ファイル名で格納**

#### 必要なフォーマット・シート
| 種類 | 名称 | 用途 |
|------|------|------|
| マニュアル | 撮影マニュアル（縦動画・ショート動画向け） | 撮影手順 |
| チェックリスト | 撮影時チェックリスト | 品質確認 |
| ツール | AIカメラアプリ | 撮影 |

### No.5 編集

| 項目 | 内容 |
|------|------|
| **担当** | 河合 |
| **次の業務** | 企業担当へ確認依頼（河合） |

#### どんな作業がありますか？

**データ取得**
1. 共有ドライブから撮影データをローカルにダウンロード
2. 元データをドライブに格納（バックアップ）

**編集作業**
1. Powerdirectorでプロジェクト作成
2. 縦型（9:16）で設定
3. 素材を読み込み
4. カット編集（不要な部分をカット、間延びした部分を調整、ジャンプカット）
5. テロップ入れ（発言内容をテキスト化、スタイル統一、表示タイミング調整）
6. BGM・SE追加（音量バランス調整、声が聞こえることが最優先）
7. 書き出し（1080x1920縦型、30fps、MP4）

**チェックリスト（書き出し前）**

テロップ誤字チェック（最重要）
- □ 人名の表記は正しいか
- □ 会社名の表記は正しいか
- □ 専門用語のスペルは正しいか
- □ 誤変換はないか

その他チェック
- □ 音声と映像の同期はずれていないか
- □ BGMの音量は適切か
- □ テロップの表示タイミングは適切か
- □ 動画の長さはショート動画として適切か（60秒以内推奨）
- □ 冒頭3秒で視聴者の興味を引く構成か

**ファイル命名規則**
```
[企業名]_[撮影日]_edit_v[バージョン].mp4
例: 中京商運_20251226_edit_v1.mp4
```

#### 何をしたら次にパスできますか？（トリガー）
**編集データを指定のドライブに指定ファイル名で格納**

## Phase 4: 運用・応募対応

### No.6 応募対応連絡（企業・応募者へ連絡）

| 項目 | 内容 |
|------|------|
| **担当** | 紺谷 |
| **次の業務** | 週間データ集計（河合）※継続運用 |

#### どんな作業がありますか？

**応募対応フロー**
```
応募受信 → 応募者情報の確認 → 応募者へ初回連絡（24時間以内）
→ 企業へ応募報告 → 面接日程調整 → 面接実施（企業側） → 結果報告
```

**Step 1: 応募受信の確認**
1. エンゲージ/Pairsonaにログイン
2. 新規応募を確認
3. 応募者情報を確認（名前、連絡先、応募動機、職歴等）

**Step 2: 応募者への初回連絡** ※24時間以内に連絡することが重要
1. 電話連絡を試みる
2. 電話がつながらない場合はメールを送信
3. 通話ログを記録

初回連絡の内容
- 応募へのお礼
- 求人内容の確認
- 面接希望日程のヒアリング
- 不明点の確認

**Step 3: 企業への報告**
1. ワークスで企業担当者に連絡
2. 応募者情報を共有
3. 面接可能日程を確認

**Step 4: 面接日程調整**
1. 応募者と企業の日程をすり合わせ
2. 確定日程を双方に連絡

### No.7 月次FB打合せ

| 項目 | 内容 |
|------|------|
| **担当** | 川崎, 下脇田 |
| **実施タイミング** | 毎月1回（キックオフMTGで確定した日程） |
| **次の業務** | 週間データ集計（河合）※継続サイクル |

#### どんな作業がありますか？

**FB承認フロー（重要）**
1. 担当者と話して実施日の決定（基本はFB実施日に次回日程を決定）
2. **実施3稼働日前まで**にFB資料の作成 → 上司提出 → 承認を得る
3. 承認を得たら、**実施1稼働日前**に企業担当者へ前日確認
4. FBの実施
5. FB時に出た課題に対しての解決は**当日内に完了**させる → 報告

**FB準備**

Step 1: データ収集
1. 月間の集計データをまとめる（閲覧数推移、イイね数推移、応募数・応募者情報）
2. 前月との比較データを作成

Step 2: FB資料作成
1. Canvaでテンプレートを開く
2. 月間データを入力
3. 推移グラフを作成
4. 考察・提案を記載

**FB実施フロー（30分想定）**

| 時間 | 内容 | ポイント |
|------|------|---------|
| 0:00-0:03 | アイスブレイク | 本音共有の場づくり |
| 0:03-0:08 | 本音ウォームアップ | 数値まだ出さない |
| 0:08-0:18 | 本音・価値観深掘り | 顧客7:Sing3 |
| 0:18-0:20 | 主観の最終確認 | 数値前の認識確認 |
| 0:20-0:23 | 閲覧数開示 | 事実のみ、評価なし |
| 0:23-0:27 | 数値の受け取り確認 | 相手の解釈を聞く |
| 0:27-0:30 | お礼・クロージング | ネクスト日程確定 |

**重要な進め方**
1. 数値は「評価」ではなく「判断材料」として扱う
2. 閲覧数は早く出さない（本音を聞いてから）
3. 本音 → 主観 → 事実 → 整理の順を守る

**FB後の対応**
1. FB報告テンプレートを使用して議事録作成
2. FBグループにテンプレートを使用して報告
3. 次回FB日程を確定しカレンダー登録

#### 何をしたら次にパスできますか？（トリガー）
**FB議事録をFBグループに共有 + ネクストFB日程の確定**
