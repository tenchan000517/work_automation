## 重要な用語定義

### 「トリガー」の概念（重要）

業務と業務の間には「トリガー」が必要。トリガー = 次の業務を開始するきっかけ・条件。

```
開始 → [トリガー] → 業務A → [トリガー] → 業務B → ... → 完了
```

例：
- 「ワークスで担当者メンション + 必要情報送信」→ 初回打ち合わせ日程調整開始
- 「議事録のワークスグループ共有」→ ヒアリング内容整理開始

### 「次の業務担当」の意味（変更あり）

**次の業務担当（nextAssignee）= 次の工程を担当する人**

- **担当**: 現在この業務を担当している人
- **次の業務担当**: この業務の次の工程を担当する人

※ 2026/01/07に「後任」から「次の業務の担当者」に意味を変更。

### Task インターフェース

```typescript
interface Task {
  no: string;              // 業務No
  category: string;        // フェーズ（旧: 大項目）
  name: string;            // 業務名（旧: 具体）
  assignee: string;        // 担当
  nextTask?: string;       // 次の業務名（動的取得も可能）
  nextAssignee: string;    // 次の担当
  tools: string;           // 使用ツール
  deliverable: string;     // 成果物
  checkpoint: string;      // チェックポイント
  hasManual: boolean;      // マニュアル有無
  issues: string;          // 備考・課題感
  // トリガー
  trigger?: string;        // 次工程へ進む条件
  // 詳細情報（オプション）
  memo?: string;           // メモ
  simulation?: string;     // シミュレーション
  manualDraft?: string;    // マニュアルたたき
  overallFlow?: string;    // 全体フロー
  format?: string;         // フォーマット
  detailedFlow?: string;   // 詳細フロー
  relatedSheetUrl?: string; // 関連シートURL（後方互換、非推奨）
  relatedLinks?: RelatedLink[]; // 複数URL対応（推奨）
}

interface RelatedLink {
  label: string;           // 表示名
  url: string;
  type: 'sheet' | 'drive' | 'site' | 'other';  // タイプ別色分け
}
```
